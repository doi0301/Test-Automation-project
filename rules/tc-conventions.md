# TC 작성 규칙 (tc-conventions.md)

> 모든 Agent가 TC를 생성할 때 따르는 표준 형식 및 채번 규칙.

---

## TC ID 채번 규칙

```
형식: {화면코드}-TC-{3자리 순번}
예시: SR01-TC-001 / SR10-TC-001 / AF07-TC-001 / E2E-TC-001
```

- 화면별 독립 채번 (SR01, SR10, AF07 각각 001부터 시작)
- E2E TC는 `E2E-TC-{3자리순번}` 독립 채번
- 결번 없이 연속 번호 사용

---

## TC 출력 형식

```markdown
### {TC-ID}: {TC 제목}

| 항목 | 내용 |
|------|------|
| Tier | T1 / T2 / T3 / T4 |
| 우선순위 | High / Medium / Low |
| 담당 화면 | {화면코드} {화면명} |
| 참조 조건 | {conditions CSV ID 또는 domain rule ID} |

**사전조건**
- {조건 1}
- {조건 2}

**실행 절차**
1. {단계 1}
2. {단계 2}
3. {단계 3}

**기대결과**
- {검증 포인트 1}
- {검증 포인트 2}
```

---

## Tier 정의

| Tier | 유형 | 설명 |
|------|------|------|
| T1 | 정상 케이스 | 주요 기능의 정상 동작 검증 |
| T2 | 예외·에러 케이스 | conditions CSV + domain WHEN-THEN 기반 예외 처리 검증 |
| T3 | 권한·역할 케이스 | permissions 매트릭스 기반 권한별 동작 검증 |
| T4 | UI·포맷 검증 | 출력 형식·레이아웃·색상 등 UI 세부 검증 |

## TC 제목 작성 규칙

```
형식: [{조건 요약}] {동작} → {기대결과 요약}
예시:
  [전문검색] 검색결과 있음 → 총 건수 및 부가기능 버튼 정상 출력
  [발명자 권한] 다운로드 버튼 → 미출력 확인
  [AF07-C08] 최대값 없는 문헌 → → 화살표 + Max 레이블 빈값 확인
```

---

## 사전조건 작성 규칙

1. **독립성 원칙**: 다른 TC 결과에 의존하지 않는 초기 상태로 기술
2. **계정 명시**: `{역할} 계정 (hipas_admin / hipas_patent01 / hipas_inventor01) 로그인 상태`
3. **환경 명시**: DEV 환경 또는 STG 환경 명시
4. **테스트 데이터 명시**: 특수 조건 데이터는 사전조건에 구체적으로 기술

---

## 기대결과 작성 규칙

1. **검증 가능하게**: "확인한다" 수준이 아닌 구체적인 출력/동작 기술
2. **부정 조건 명시**: 미출력·미제공 케이스는 "X가 출력되지 않는다" 명확히 기술
3. **수치 기반**: 소수점, 건수, 색상 등은 기대값을 구체적으로 기술

---

## 금지 사항

- `TC-000` 형식 사용 금지 (001부터 시작)
- 연속 번호 건너뜀 금지
- "이전 TC 결과 상태" 사전조건 사용 금지 → 독립성 위반
- forbidden-patterns.md 항목에 해당하는 TC 작성 금지
