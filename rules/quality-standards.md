# TC 품질 기준 (quality-standards.md)

> TC 생성 완료 후 모든 Agent와 팀 리드가 적용하는 품질 3원칙 + 체크리스트.

---

## 품질 3원칙

### 원칙 1 — 독립성 (Independence)
각 TC는 독립된 초기 상태에서 시작해야 한다.
- **위반 사례**: "SR01-TC-001이 Pass된 상태에서 진행"
- **올바른 작성**: 사전조건에 초기 상태를 직접 기술

### 원칙 2 — 재현 가능성 (Reproducibility)
실제 환경에서 재현 가능한 사전조건을 사용해야 한다.
- **위반 사례**: "검색결과 10만 건이 있는 상태" (DB 데이터 보장 불가)
- **올바른 작성**: "검색결과 N건 이상 확인된 상태" 또는 테스트 데이터 준비 명시

### 원칙 3 — 커버리지 완전성 (Coverage Completeness)
conditions/ CSV의 모든 조건 행이 TC로 생성되어야 한다.
- **측정**: conditions/ CSV `tc_generated: false` 잔여 행 = 0
- **예외**: `allowed: false` 전이 → TC 대상 제외

---

## 품질 게이트 체크리스트 (7항목)

TC 생성 완료 후 아래 전항목을 순서대로 실행한다.

```
□ 체크 1: conditions/ CSV tc_generated: false 잔여 행 = 0
          → 잔여 있으면 해당 조건 TC 추가 생성

□ 체크 2: state-transitions.csv allowed: true + tc_generated: false = 0
          → 잔여 있으면 해당 전이 TC 추가 생성

□ 체크 3: domain-rules.md MUST NOT 해당 TC = 0
          → 위반 TC 발견 시 즉시 삭제

□ 체크 4: 권한 3종 커버리지 — 관리자/특허담당자/발명자 각 1개 이상 TC 존재
          → 누락된 역할 TC 추가 생성

□ 체크 5: 합금 도메인 규칙 TC 3개 필수 존재
          - AF07-C08 (최대값 없는 문헌 → 화살표 처리)
          - AF07-C09 (최소값 없는 문헌 → 화살표 처리)
          - AF07-C13 (미추출 문헌 Y축 제외 + 하단 목록)

□ 체크 6: SR10 소수점 출력 포맷 검증 TC 1개 이상 존재
          → 소수점 임의 변환 금지(MUST-01) 검증 TC

□ 체크 7: E2E 연결 TC 1개 이상 (SR01 → SR10 → AF07 흐름)
          → agent-lead 단독 작성 E2E-TC-001 이상
```

---

## 품질 게이트 결과 리포트 형식

```
품질 게이트 통과: {통과수}/7
총 TC: N개 (SR01: N / SR10: N / AF07: N / E2E: N)
커버리지: 조건 {N}% / 권한 3종 / 도메인 규칙 {N}개
미통과 항목: {해당 체크 번호 + 조치 내용}
```
