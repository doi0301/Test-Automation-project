## TC 생성 핵심 포인트

### FZ016 (연차정보 입력 필수값) — data/FZ016-fields.json 참조

필수값 미입력 시 접수조건 N 처리:
- 이관연차, 이관연차납부마감일, PTA, 공동출원인 필수 여부 확인
- 각 필드 미입력/오입력 케이스 TC

### FZ017 (접수) — 접수 프로세스 핵심

플로우차트 기반 전체 흐름:
- 정상 접수 → 신건 발수값 TASK uid 전송 확인
- 접수 후 상태 "접수중" 전환 확인
- 연차건 KEY 생성 확인 (Mock 필요)

### FZ021 (신건확인) — 반려 분기 3경로 모두 TC 필수

| 경로 | 결과 |
|------|------|
| 정상 확인 | 접수완료 전환 |
| 반려(긴급) | 접수취소 전환 + 메일 발송 |
| 반려(중복) | 접수취소 전환 + 메일 발송 |
| 공동고객 접수취소 | 접수취소 전환 (메일 없음) |
